# 1. What is a Blockchain?

### Definition (Technical):

A blockchain is a:

- Shared (everyone has a copy)
    
- Immutable (cannot be changed)
    
- Ledger (ordered record book)
    

It records:

- Transactions
    
- Assets
    
- State changes
    

It builds trust by using cryptography + distributed consensus.

---

# 2. How Blockchain Stores Data (Blocks)

A blockchain is a sequence of blocks, each containing:

### 2.1 What a Block Contains

Each block typically has:

1. Previous Block Hash
    
    - A cryptographic hash of the _entire previous block_.
        
    - This links blocks together.
        
2. Timestamp
    
    - When this block was created / last modified.
        
3. Transactions (or data)
    
    - Each transaction also has a hash.
        
    - All transaction hashes roll up into a Merkle root.
        

### 2.2 Merkle Tree (Technical)

- Each transaction â†’ hashed
    
- Pairs of hashes â†’ combined and hashed
    
- This repeats until 1 final hash exists â†’ Merkle Root
    

This ensures:

- Integrity (if any transaction changes, Merkle root changes)
    
- Efficient verification (you don't need the whole dataset)
    

---

# 3. Why Each Block Stores the Previous Hash

If block 11 stores the hash of block 10:

- Changing block 10 â†’ its hash changes â†’ block 11's "previous hash" becomes invalid â†’ the entire chain breaks.
    

This creates cryptographic immutability.

You cannot secretly modify any old record.

# 4. Public Ledger (Technical)

A public ledger is:

- A distributed record-keeping system
    
- Maintained across thousands of nodes
    
- Identities are cryptographically protected  
    (public key addresses, not real names)
    
- Stores:
    
    - balances
        
    - transactions
        
    - state history
        

### No centralized server.

Every node stores its own copy.

# 6. Decentralization (Technical)

Decentralized means:

- No single authority controls the ledger.
    
- Nodes validate data through consensus algorithms (e.g., Proof of Work).
    
- Even if some nodes are malicious, the chain stays valid if the majority is honest.

# 7. Smart Contracts (Technical)

### Definition

A smart contract is code stored on a blockchain that automatically executes when conditions are met.

Technical characteristics:

- Written in code (e.g., Solidity on Ethereum)
    
- Self-executing (no human needed)
    
- Deterministic (same input â†’ same output)
    
- Immutable once deployed (cannot be changed)
    
- Transparent (anyone can see it)
    

### Example

"When buyer confirms delivery, transfer funds to seller."

This logic is encoded directly into the contract.

### Benefits

- Removes intermediaries (banks, lawyers)
    
- Reduces fraud
    
- Automates workflows
    
- Every action is written permanently to the blockchain ledger
