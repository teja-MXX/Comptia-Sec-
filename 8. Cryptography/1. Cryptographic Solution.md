## 1. What is Cryptography?

Cryptography = the science of writing codes + solving codes so the real meaning is hidden.  
Modern cryptography mainly = encryption.

### Encryption

- Converts plaintext ‚Üí ciphertext (unreadable).
    
- Only someone with the correct key can turn ciphertext back ‚Üí plaintext.
    

Encryption protects data in all 3 states:

- Data at rest (stored on HDD/SSD).
    
- Data in transit (moving over network or memory bus).
    
- Data in use (being processed inside CPU).
    

---

# 2. 3 States of Data

### Data at Rest

- Inactive, stored on disk.
    
- Needs full-disk encryption or file-level encryption.
    

### Data in Transit

- Moving over a network, or moving between RAM ‚Üî CPU ‚Üî storage.
    
- Needs protocols like TLS, SSH, IPsec.
    

### Data in Use

- Currently being processed or modified by CPU.
    
- Hardest to protect; usually protected via secure enclaves, memory encryption.
    

---

# 3. Basic Encryption Example: ROT13

ROT13 is a cipher (algorithm).  
Key = 13 (rotate letters by 13 positions).

Important point:  
‚û° Security does NOT come from the algorithm ‚Äî it comes from the key.

Modern algorithms are:

- public
    
- peer-reviewed
    
- strength = key secrecy + key length
    

---

# 4. What is a Cipher/Algorithm?

A cipher = mathematical function that transforms data.  
Input ‚Üí (math operation) ‚Üí output.

ROT13 example:

- Input: C
    
- Apply algorithm: rotate 13 letters
    
- Output: P
    

All encryption works like this but MUCH more complex than ROT13.

---

# 5. Importance of Keys

Encryption strength = key length, not the algorithm.

üìå Key length ‚àù exponential security  
Example:

- 128-bit key ‚Üí strong
    
- 256-bit key ‚Üí exponentially stronger (NOT double, but squared effect)
    

### Key Rotation

Keys must be:

- long
    
- secret
    
- rotated regularly
    

Why?

- Long exposure = risk of guessing/bruteforce
    
- Technology improves (faster cracking)
    

Typical rotations:

- TLS certificates ‚Üí yearly
    
- Cloud KMS systems ‚Üí ~90 days (automatic)
    

---

# 6. Algorithms Are Public

Very important idea:

- Algorithms are NOT SECRET.
    
- They are public & open source.
    
- Security = secrecy of the key, NOT obscurity (security-through-obscurity = bad).
    

Keys must be:

- stored in secure modules (HSMs, TPMs)
    
- encrypted when stored
    
- transmitted securely
    
- access tightly restricted
    
- audited regularly
    

If the key is exposed ‚Üí the entire system collapses.

# 8. Topics Covered (Detailed Syllabus)

## A. Symmetric vs Asymmetric Algorithms

### Symmetric Encryption

- One single key
    
- Same key for encryption & decryption
    
- Faster
    
- Used for bulk data encryption
    

Algorithms:

- DES
    
- 3DES
    
- IDEA
    
- AES
    
- Blowfish
    
- Twofish
    
- RC (Rivest Cipher family)
    

---

### Asymmetric Encryption

- Two keys:
    
    - Public key (encrypt)
        
    - Private key (decrypt)
        
- Slower
    
- Used for key exchange, digital signatures
    

Algorithms:

- Diffie-Hellman
    
- RSA
    
- Elliptic Curve Cryptography (ECC)
    

---

## B. Hashing

Hash = fixed-size output from any input.  
Used for:

- integrity
    
- password storage
    
- signature creation
    

Algorithms:

- MD5
    
- SHA-1 / SHA-2 / SHA-3
    
- RIPEMD
    
- HMAC (hash + key)
